# 基于 FPGA 的 LM75A 智能温度监测报警系统

##  一、项目市场概况与技术价值
温度监测是**工业控制、服务器运维、智能家居、电子仪器**中不可或缺的基础环节。传统单片机方案在处理高频采样、多通道并行、复杂时序联动时存在**调度开销大、实时性弱**等缺陷。

本项目采用 **FPGA 纯硬件并行架构** 实现，具有**极低延迟（纳秒级）、高抗干扰、高时序可控**的优势，完美适配温度这类对实时性要求极高的物理量测量。

**项目价值定位**：
1. **教学展示**：深度覆盖 I2C 协议、补码运算、动态扫描、状态机等核心数字电路知识点。
2. **工程示范**：实现了从传感器采集到数据展示再到报警执行的**全链路工业级流程**。
3. **实用价值**：可快速部署为小型环境监测盒、实验室温控节点或教学实验箱演示平台。

---

##  二、项目整体架构与层次划分
本项目严格遵循 **自顶向下、模块化、高内聚低耦合** 的 FPGA 设计准则，共包含 **7 个 Verilog 模块**，分为**采集核心、处理核心、控制执行、人机交互**四大层级。

### 系统层级架构图
```
┌─────────────────────────────────────────────────────┐
│                 顶层模块：LM75A_Top                │  ← 系统总控
└───────────────────┬───────────────────────────────┘
                    │
┌───────────────────┴───────────────────────────────┐
│  【数据采集层】                                     │
│  Control_Unit (I2C 主控) + Button_Debounce (消抖)   │  ← 核心输入层
└───────────────────┬───────────────────────────────┘
                    │
┌───────────────────┴───────────────────────────────┐
│  【数据处理层】                                     │
│           Data_Processor (补码/BCD/温标转换)        │  ← 逻辑核心
└───────────────────┬───────────────────────────────┘
                    │
┌───────────────────┴───────────────────────────────┐
│  【输出执行层】                                     │
│  Alarm_System (报警) + Seg_Driver (显示) +          │
│  RGB_Indicator (状态指示)                          │  ← 核心输出层
└─────────────────────────────────────────────────────┘
```

##  三、核心模块技术解析
### 模块一：Control_Unit（I2C 主控单元）
> **定位**：系统数据来源，所有温度数据均由此模块生成。

####  核心技术难点与源码分析
*   **10 状态全流程 I2C 主机实现**
    ```verilog
    localparam IDLE=0, START=1, SEND_ADDR=2, ACK_1=3, READ_MSB=4, ACK_2=5, READ_LSB=6, NACK=7, STOP=8, DONE=9;
    ```
    覆盖了从**起始(Start)到停止(Stop)** 的每一个步骤，任何一个状态跳转偏差都会导致 NACK（无应答）。

*   **4 相节拍精准时序控制**
    I2C 协议要求 SDA 数据必须在 SCL 低电平期间变化，高电平期间稳定。将一个 SCL 周期拆分为 4 个相位（step 0-3），严格控制每一步的电平翻转：
    ```verilog
    // 以发送地址为例，SDA 只能在 SCL=0 时改变
    if (step == 0) begin sda_out <= LM75A_RD_CMD[7 - bit_cnt]; scl_out <= 1'b0; end
    if (step == 1) begin scl_out <= 1'b1; end // SCL 拉高，此时数据必须稳定
    ```

*   **传感器在线检测与容错**
    在 ACK_1 阶段加入了关键的在线检测逻辑，保证系统稳定性：
    ```verilog
    if (sda_in == 1'b1) state <= STOP; // 检测到 SDA 未拉低，强制终止通信
    ```

*   **高精度数据截取**
    LM75A 传回 16 位数据，准确截取了高 12 位作为有效温度：
    ```verilog
    temp_raw <= rx_data[15:4];
    ```

### 模块二：Data_Processor（数据处理单元）
> **定位**：将原始二进制数据转换为人眼可读的 BCD 码。

####  核心技术难点与源码分析
*   **补码符号位扩展与溢出保护**
    为了在摄氏度转华氏度时防止数据溢出，进行了安全的位扩展：
    ```verilog
    wire signed [15:0] signed_c = { {4{temp_raw[11]}}, temp_raw };
    ```

*   **硬件化温标转换（C ↔ F）**
    不使用浮点数运算，纯整数运算实现精确转换：
    ```verilog
    wire signed [15:0] signed_f = (signed_c * 14'sd9) / 14'sd5 + 14'sd512;
    ```

*   **补码求绝对值（原码转换）**
    ```verilog
    wire [15:0] abs_val = active_temp[15] ? (~active_temp + 1'b1) : active_temp;
    ```

*   **小数位 BCD 编码技巧**
    LM75A 低 4 位代表 1/16℃，通过 `(*10) >> 4` 实现权重转换：
    ```verilog
    bcd_dec <= (frac_part * 10) >> 4;
    ```
### 模块三：Alarm_System（报警系统单元）
> **定位**：系统的“大脑”输出，负责声光报警。

####  核心技术难点与源码分析
*   **工业级连续采样滤波（10 次确认）**
    为了防止温度抖动导致误报，没有采用单次判断，而是设计了**级联防抖逻辑**：
    ```verilog
    if (confirm_cnt < 4'd10) confirm_cnt <= confirm_cnt + 1'b1;
    else alarm_active <= 1'b1;
    ```

*   **2Hz LED 闪烁分频**
    从 50MHz 分频到 0.25 秒翻转一次，逻辑严谨：
    ```verilog
    if (led_cnt >= 24'd12_500_000) begin ... end
    ```

*   **2kHz 蜂鸣器 PWM 生成**
    实现了占空比 50% 的方波驱动：
    ```verilog
    buzzer_pwm <= (pwm_cnt < 15'd12500);
    ```


### 模块四：Seg_Driver（数码管驱动单元）
> **定位**：高精度可视化输出。

####  核心技术难点与源码分析
*   **6 位动态扫描与时分复用**
    利用人眼视觉暂留效应，轮流点亮 6 个数码管：
    ```verilog
    parameter SCAN_MAX = 50_000; // 1ms 扫描周期
    ```

*   **消重影机制（关键优化）**
    在切换数码管瞬间，强行短暂全灭，解决硬件延迟导致的重影问题：
    ```verilog
    if (scan_cnt < 10) begin seg_sel <= 6'b111111; seg_led <= 8'h00; end
    ```

*   **智能灭零（首零消隐）**
    只显示有效数字，高位多余的 0 自动熄灭：
    ```verilog
    wire blank_high = (bcd_high == 4'd0);
    wire blank_mid  = (blank_high) && (bcd_mid == 4'd0);
    ```

### 模块五：Button_Debounce（按键消抖单元）
> **定位**：人机交互接口。

#### 1. 核心技术难点
*   **20ms 稳态检测**
    过滤机械按键抖动，输出干净的单脉冲信号。

*   **跨时钟域同步**
    ```verilog
    always @(posedge clk) btn_sync <= {btn_sync[0], btn_in};
    ```

---

### 模块六：RGB_Indicator（RGB 指示灯单元）
> **定位**：系统状态可视化。

####  核心技术难点
*   **三态互斥逻辑**
    实现待机（蓝）、正常（绿）、报警（红）三种状态的互斥控制。

---

### 模块七：LM75A_Top（顶层模块）
> **定位**：胶水逻辑，系统总控。

####  核心技术难点
*   **多模块互联与顶层设计**
    例化 6 个子模块，处理复杂的总线互联（如 inout 端口的处理）和状态控制逻辑。

*   **待机模式与全局逻辑屏蔽**
    在顶层实现待机逻辑，在复位时屏蔽报警和采集，实现系统的低功耗管理。

##  四、综合技术复杂度总结（答辩/评分核心亮点）
本项目在以下 5 个维度展现了极高的技术水平，远超普通课程设计要求：

### 1. 通信协议实现深度（Control_Unit）
*   **非 IP 依赖**：纯手写 I2C 状态机，而非调用 IP 核。
*   **时序严谨**：精准控制 SCL/SDA 四相节拍，无毛刺通信。
*   **容错机制**：加入在线检测与自动终止逻辑，工业级鲁棒性。

### 2. 有符号数与定点运算深度（Data_Processor）
*   **补码处理**：熟练掌握补码扩展、求绝对值、符号位扩展。
*   **硬件转换**：用整数运算实现高精度摄氏度/华氏度转换，无浮点数资源消耗。

### 3. 多级时序协同设计（全项目）
*   **多周期并行**：在 50MHz 主时钟下，同时运行 I2C(100k)、消抖(20ms)、扫描(1ms)、PWM(2kHz) 等多种不同频率的时序逻辑。
*   **数据握手**：严格依赖 `data_vld` 脉冲信号，实现跨模块数据同步，无亚稳态风险。

### 4. 外设驱动与混合信号设计（Alarm_System, Seg_Driver）
*   **PWM 生成**：纯逻辑实现数模转换，驱动蜂鸣器。
*   **动态扫描**：时分复用技术，解决数码管位数限制与硬件引脚限制。

### 5. 工程化优化与抗干扰设计
*   **温度防抖**：10 次连续采样确认，杜绝误报。
*   **显示消影**：切换瞬间全灭，解决重影问题。
*   **按键消抖**：标准 20ms 防抖与跨时钟域同步。

##  五、系统功能与端口说明
### 核心功能清单
1.  **精准采集**：LM75A 12bit 补码采集，精度 0.0625℃。
2.  **双温标切换**：摄氏度/华氏度自由转换。
3.  **智能报警**：0℃~50℃ 超温检测，连续 10 次异常触发。
4.  **声光联动**：LED 闪烁 + 蜂鸣器鸣叫 + RGB 红灯。
5.  **状态可视**：RGB 灯指示待机/正常/报警。

### 顶层端口说明
| 端口名 | 方向 | 位宽 | 功能描述 |
|:--- |:--- |:--- |:--- |
| **clk** | 输入 | 1 | **50MHz** 系统时钟输入 |
| **rst_n** | 输入 | 1 | 低电平有效复位 |
| **btn_mode** | 输入 | 1 | 温标切换按键（0=℃, 1=℉） |
| **btn_standby** | 输入 | 1 | 待机/工作模式切换按键 |
| **sda** | inout | 1 | I2C 数据总线（三态门） |
| **scl** | 输出 | 1 | I2C 时钟总线（100kHz） |
| **rgb_b/g/r** | 输出 | 3 | RGB 状态灯（蓝/绿/红） |
| **seg_sel[5:0]** | 输出 | 6 | 6 位数码管位选（低电平选中） |
| **seg_led[7:0]** | 输出 | 8 | 数码管段选与小数点 |
| **led_alarm** | 输出 | 1 | 报警 LED 指示灯 |
| **buzzer_pwm** | 输出 | 1 | 蜂鸣器 PWM 驱动输出 |
